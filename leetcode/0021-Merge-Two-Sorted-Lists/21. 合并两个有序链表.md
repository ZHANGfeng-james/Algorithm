[21. 合并两个有序链表 - 力扣（LeetCode） (leetcode-cn.com)](https://leetcode-cn.com/problems/merge-two-sorted-lists/)

将两个**升序链表**合并为一个新的 **升序** 链表并返回。新链表是通过**拼接**给定的两个链表的**所有节点**组成的。 

**示例 1：**

![](./Snipaste_2021-09-06_08-28-33.png)

~~~go
输入：l1 = [1,2,4], l2 = [1,3,4]
输出：[1,1,2,3,4,4]
~~~

**示例 2：**

```go
输入：l1 = [], l2 = []
输出：[]
```

**示例 3：**

```go
输入：l1 = [], l2 = [0]
输出：[0]
```

**提示：**

- 两个链表的节点数目范围是 `[0, 50]`
- `-100 <= Node.val <= 100`
- `l1` 和 `l2` 均按 **非递减顺序** 排列

# 解题思路

在代码部分，有如下提示：

其中定义的**单链表结构**是这样的：

~~~go
type ListNode struct {
	Val  int
	Next *ListNode
}
~~~

暴力解决办法（**链表**的解决办法）：

~~~go
package leetcode

import "fmt"

func (list *ListNode) printAll() {
	for list != nil {
		fmt.Printf("node:%d\n", list.Val)
		list = list.Next
	}
}

func mergeTwoLists(l1 *ListNode, l2 *ListNode) *ListNode {
	if l1 == nil {
		return l2
	}
	if l2 == nil {
		return l1
	}

	mergeList := &ListNode{}
	flagPtr := mergeList
	for l1 != nil && l2 != nil {
		if l1.Val >= l2.Val {
			newNode := &ListNode{
				Val:  l2.Val,
				Next: nil,
			}
			flagPtr.Next = newNode
			flagPtr = flagPtr.Next

			l2 = l2.Next
		} else {
			newNode := &ListNode{
				Val:  l1.Val,
				Next: nil,
			}
			flagPtr.Next = newNode
			flagPtr = flagPtr.Next

			l1 = l1.Next
		}
	}

	for l1 != nil {
		newNode := &ListNode{
			Val:  l1.Val,
			Next: nil,
		}
		flagPtr.Next = newNode
		flagPtr = flagPtr.Next
		l1 = l1.Next
	}

	for l2 != nil {
		newNode := &ListNode{
			Val:  l2.Val,
			Next: nil,
		}
		flagPtr.Next = newNode
		flagPtr = flagPtr.Next
		l2 = l2.Next
	}

	return mergeList.Next
}
~~~

另外的一个思路：借鉴**插入排序**的思路

1. 把第一个单链表作为**已排序区**，第二个链表作为**未排序区**。每次循环时，从未排序区中选择首个元素，插入已排序区中；
2. 仅需要 3 个 ListNode 的指针，分别指向 l1 和 l2，以及最终合并的链表。

依据这个思路得到的实现代码如下：

~~~go
/**
 * Definition for singly-linked list.
 * type ListNode struct {
 *     Val int
 *     Next *ListNode
 * }
 */
func mergeTwoLists(l1 *ListNode, l2 *ListNode) *ListNode {
	if l1 == nil {
		return l2
	}
	if l2 == nil {
		return l1
	}

	// l1作为已排序区，l2作为未排序区
	mergeList := &ListNode{}
	result := mergeList
	for l1 != nil && l2 != nil {
		if l1.Val > l2.Val {
			mergeList.Next = l2

			tmp := l2.Next
			l2.Next = nil
			l2 = tmp
		} else {
			// l1.Val <= l2.Val 取 l1.Val
			mergeList.Next = l1

			tmp := l1.Next
			l1.Next = nil
			l1 = tmp
		}
		mergeList = mergeList.Next
	}

	if l1 != nil {
		mergeList.Next = l1
	}

	if l2 != nil {
		mergeList.Next = l2
	}

	return result.Next
}
~~~

上面是 2 种不同的实现方式，但解决办法的思路是相同的，都是比较。从代码实现来看，l1.Val > l2.Val 或者 l1.Val <= l2.Val 情况中，代码结构是相同的。这是很重要的一个特征！

这个题目和**归并排序**中的递归有相类似的思路。我们可以如下递归地定义两个链表里的 `merge` 操作（忽略边界情况，比如空链表等）：
$$
\begin{cases} 
list1[0] + merge(list1[1:], list2),  & list1[0] < list2[0] \\
list2[0] + merge(list1, list2[1:]), & \text otherwise
\end{cases}
$$
也就是说，两个链表头部值较小的一个节点与剩下元素的 `merge` 操作结果合并。

我们直接**将以上递归过程建模**，同时需要考虑边界情况。

如果 l1 或者 l2 一开始就是空链表 ，那么没有任何操作需要合并，所以我们只需要返回非空链表。否则，我们要判断 l1 和 l2 哪一个链表的头节点的值更小，然后递归地决定下一个添加到结果里的节点。如果两个链表有一个为空，递归结束。

~~~go
func mergeTwoLists(l1 *ListNode, l2 *ListNode) *ListNode {
	if l1 == nil {
		return l2
	}
	if l2 == nil {
		return l1
	}

	if l1.Val < l2.Val {
		// “递”，本次取得 l1 作为当前处理的结果（“归”），取 l1
		l1.Next = mergeTwoLists(l1.Next, l2)
		return l1
	} else {
		// l1.Val >= l2.Val，取 l2
		// “递”，本次取得 l2 作为当前处理的结果（“归”），取 l2
		l2.Next = mergeTwoLists(l1, l2.Next)
		return l2
	}
}
~~~

func createList(nums []int) *ListNode {
	l1 := &ListNode{}
	head1 := l1
	for _, ele := range nums {
		newNode := &ListNode{
			Val:  ele,
			Next: nil,
		}

		newNode.Next = l1.Next
		l1.Next = newNode

		l1 = l1.Next
	}
	head1 = head1.Next
	return head1
}
